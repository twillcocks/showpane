# Showpane Generated Preamble

This file is documentation for maintainers.

The actual shared preamble injected into skills is generated by:

- `packages/cli/scripts/gen-skill-docs.mjs`
- `packages/cli/scripts/lib/skill-docs.mjs`

Do not hand-copy this file into `skills/portal-*/SKILL.md`.

## What the generated preamble owns

- update checks
- global config reads from `~/.showpane/config.json`
- session tracking in `~/.showpane/sessions/`
- telemetry/session bookkeeping in `~/.showpane/analytics/`
- one-time telemetry prompt
- recent skill / learnings discovery
- shared next-skill routing hints
- shared runtime-principles / platform-constraints reminders

## Runtime files

The generated preamble assumes these user-local paths:

- `~/.showpane/config.json`
- `~/.showpane/bin/showpane-config`
- `~/.showpane/bin/showpane-update-check`
- `~/.showpane/bin/showpane-telemetry-log`
- `~/.showpane/current/`
- `~/.showpane/timeline.jsonl`
- `~/.showpane/learnings.jsonl`
- `~/.showpane/checkpoints/`

## Editing guidance

- Change shared preamble behavior in the generator/resolver layer, not in generated `SKILL.md` files.
- Keep workflow-specific logic in `SKILL.md.tmpl`.
- Run `npm run gen:skill-docs` in `packages/cli` after editing templates or resolver code.
- Run `npm run skill:check` in `packages/cli` before shipping.
